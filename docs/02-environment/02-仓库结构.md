# 仓库结构

## 1. 结构目标

仓库按“代码、基础设施、文档”三条线分开，避免后期目录失控。

## 2. 根目录规划

```text
Personal_Projections/
├─ docs/
├─ infra/
├─ local-life-server/
├─ local-life-copilot/
├─ copilot-agent-service/
├─ performance-tests/
├─ scripts/
├─ pom.xml
├─ README.md
└─ .gitignore
```

## 3. 各目录职责

### `docs/`

存放全过程文档，包括：

- `01-project/`：项目蓝图、边界、架构、路线
- `02-environment/`：环境与基础设施文档
- `03-process/`：Git 与协作规范
- `04-notes/`：深度教程、测试总结、面试题库
- `demo/`：面试演示脚本

核心内容包括：

- 环境搭建
- 项目边界
- 领域模型
- ER 图
- 时序图
- 技术选型
- 缓存设计
- MQ 与一致性
- Agent/RAG/MCP 设计
- 测试与压测
- 故障演练
- 复盘与简历表达

### `infra/`

存放本地开发和后续部署所需基础设施：

- `docker-compose.dev.yml`
- 容器环境变量模板
- 初始化 SQL
- 监控配置

### `local-life-server/`

传统后端主项目模块，承载门店、内容、社交、营销、交易、搜索、对账补偿等系统能力。

### `local-life-copilot/`

Java MCP Server，把业务能力封装成 Agent 可调用工具。

### `copilot-agent-service/`

Python Agent Service，承载 FastAPI、LangGraph/ReAct、Tool Router、RAG、HITL、Evals。

## 4. 模块边界

### 应用模块边界

- `local-life-server`：业务事实来源，负责真实交易状态。
- `local-life-copilot`：工具网关，负责 MCP 协议、RBAC、审计、限流。
- `copilot-agent-service`：编排层，负责 LLM、Fast Path、ReAct、RAG、HITL。

## 5. 当前目录策略

- 主学习入口从 `docs/00-学习路线.md` 开始。
- 查代码入口看 `docs/01-project/02-代码级实现地图.md`。
- 查接口看 Swagger 和 `docs/01-project/10-接口规范文档.md`。
