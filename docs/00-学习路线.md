# LocalLife 学习路线

这份文档是整个 `docs/` 的主入口。阅读目标不是背文档，而是能把一个需求讲到接口、代码、表结构、中间件、测试和面试追问。

## 0. 先建立运行感

先跑起来，再看设计。否则容易把项目看成一堆名词。

1. 启动服务：看 [README.md](../README.md) 的 Docker Compose 启动方式。
2. 打开接口文档：看 [Swagger 在线接口文档](./04-notes/Swagger在线接口文档.md)。
3. 灌演示数据：`bash scripts/business-bulk-simulate.sh`。
4. 按 [面试演示脚本](./demo/面试演示脚本.md) 跑一遍 MCP 查询和 Agent 对话。

跑通后你应该能说清楚：

- Java 主服务、Java MCP 服务、Python Agent 服务分别做什么。
- 为什么 Agent 不直接读业务库，而是通过 MCP 工具调用受控能力。
- 当前真实数据从 MySQL/Redis 进来，怎么被 Swagger、MCP 和 Agent 读到。

## 1. 从需求和边界开始

阅读顺序：

1. [项目边界文档](./01-project/03-项目边界文档.md)
2. [领域模型文档](./01-project/04-领域模型文档.md)
3. [ER 图文档](./01-project/05-ER图文档.md)

这一阶段只解决三个问题：

- 这个项目到底做什么业务，不做什么业务。
- 用户、商家、门店、笔记、券、订单、支付之间是什么关系。
- 哪些表是主链路核心表，哪些只是辅助表。

面试常问：

- 为什么要把订单和支付单拆开？
- 商家、门店、用户之间怎么建模，为什么不是一张大表？
- 为什么优惠券要拆成券模板、秒杀场次、用户券？

## 2. 从接口规范进入后端实现

阅读顺序：

1. [接口规范文档](./01-project/10-接口规范文档.md)
2. [LocalLife 项目接口教程](./04-notes/LocalLife项目接口教程.md)
3. [LocalLife Server 代码关系图](./04-notes/LocalLifeServer代码关系图.md)
4. [代码级实现地图](./01-project/02-代码级实现地图.md)
5. [核心时序图文档](./01-project/06-核心时序图文档.md)

这一阶段要按“接口入口 → Controller → Service → Mapper/Entity → Redis/MQ/ES → 测试”去读代码。

重点链路：

- 登录鉴权：验证码、登录、Token、拦截器、ThreadLocal。
- 门店缓存：Caffeine + Redis + BloomFilter + Canal/binlog 思路。
- 笔记互动：点赞、评论、共同关注、Redis ZSet。
- 秒杀抢券：Redis Lua 原子扣减、防重复、DB 唯一索引兜底。
- 订单支付：幂等下单、支付回调、延迟关单、Transactional Outbox。
- 搜索与限流：ES 检索、Redis Lua 滑动窗口。

面试常问：

- Redis Lua 为什么能防超卖？
- 缓存一致性为什么不用“更新缓存”，而是删除缓存？
- 支付回调为什么必须幂等？
- Outbox 相比直接发 MQ 解决了什么问题？
- 订单分片为什么选 `user_id % 4`？

## 3. 再看 Agent 和 MCP

阅读顺序：

1. [后端与 Agent 链路架构图](./01-project/12-后端与Agent链路架构图.md)
2. [Copilot 企业级 Agent 设计](./01-project/07-Copilot企业级Agent设计.md)
3. [LocalLife Copilot 项目教程](./04-notes/LocalLifeCopilot项目教程.md)
4. [RAG 模型选型说明](./01-project/RAG模型选型说明.md)
5. [模型解耦设计](./01-project/模型解耦设计.md)

这一阶段要理解三层边界：

- Python Agent 负责对话、推理、RAG、HITL、状态持久化。
- Java MCP Server 负责工具协议、RBAC、审计、限流、业务工具封装。
- Java 主业务服务负责订单、支付、退款、补券等真实业务规则。

面试常问：

- 为什么不让 LLM 直接调用 Java REST 接口？
- MCP Server 比普通 Controller 多了什么价值？
- Fast Path 和 ReAct 分别适合什么场景？
- HITL 为什么需要 checkpoint，而不是只存聊天记录？
- RAG 为什么要有 reranker，为什么要做权限过滤？

## 4. 工程化和质量门禁

阅读顺序：

1. [测试总览与结果汇总](./04-notes/测试总览与结果汇总.md)
2. [企业级测试实践：集成测试与压测](./04-notes/企业级测试实践-集成测试与压测.md)
3. [企业级日志系统](./04-notes/企业级日志系统.md)
4. [Git 版本管理与提交规范](./03-process/Git版本管理与提交规范.md)

你要能讲出测试分层：

- 单元测试：Service/Agent 节点/Guardrails/Tool Router。
- 集成测试：Testcontainers 真 MySQL/Redis。
- 契约测试：Agent MCP Client 和 Java MCP Provider 的协议契约。
- E2E/Smoke：Agent → MCP → DB。
- 覆盖率和变异测试：JaCoCo、pytest-cov、PIT、mutmut。
- 压测：Locust/k6 验证秒杀和核心链路。
- 日志与追踪：Loki/Promtail/Grafana、traceId、审计表、日志告警。

面试常问：

- 为什么不用 H2，而用 Testcontainers？
- 覆盖率为什么不是越高越好？
- 变异测试比行覆盖多验证了什么？
- 秒杀压测怎么判断“没有超卖”？

## 5. 最后准备面试表达

阅读顺序：

1. [面试演示脚本](./demo/面试演示脚本.md)
2. [后端 + Agent 面试高频题库](./04-notes/面试高频题库-后端Agent.md)
3. [深度拷打与面试指南](./04-notes/深度拷打与面试指南.md)

推荐表达顺序：

1. 先说业务：本地生活交易平台。
2. 再说后端：高并发、缓存、秒杀、支付一致性、分表、ES、MQ。
3. 再说 Agent：MCP 工具、RBAC、HITL、RAG、Evals。
4. 最后说工程化：Docker、CI、测试门禁、压测、可观测。

不要一开口就堆技术名词。每个技术点都要绑定一个业务问题：

| 技术 | 解决的业务问题 |
| --- | --- |
| Redis Lua | 秒杀库存原子扣减、防超卖 |
| Outbox | 支付成功事件不丢、可重试 |
| ShardingSphere | 用户订单列表单分片命中 |
| MCP | 让 Agent 安全调用业务能力 |
| HITL | 退款、补券等高风险动作必须人工审批 |
| RAG | 平台规则和排障案例可检索、可引用 |
| Testcontainers | 用真实 MySQL/Redis 验证 Mock 测不到的行为 |

## 建议的学习节奏

第一天：跑通服务、Swagger、批量数据、演示脚本。

第二天：读需求、领域模型、ER 图，把所有核心表手画一遍。

第三天：读 Java 主链路，重点是登录、门店缓存、秒杀、订单支付。

第四天：读 MCP 和 Python Agent，重点是工具调用、Fast Path、ReAct、HITL。

第五天：读测试和深度面试指南，整理自己的 8 分钟讲稿。

第六天以后：按 [代码级实现地图](./01-project/02-代码级实现地图.md) 回到代码里逐个断点/打日志验证。
