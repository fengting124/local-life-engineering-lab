# 模型解耦设计

> 核心原则：Agent Service 不直接 import 任何 ML 模型，所有模型调用通过 HTTP 服务。

---

## 架构图

```
copilot-agent-service
       │
       ├── rag/embedding_client.py  ──HTTP──▶  embedding-service:8100
       │                                         (sentence-transformers)
       ├── rag/reranker_client.py   ──HTTP──▶  reranker-service:8101
       │                                         (CrossEncoder)
       └── rag/vector_store.py     ──gRPC──▶  Milvus:19530
```

---

## 设计决策

### 为什么不在 Agent Service 里直接 import 模型？

1. **避免重量级依赖污染**：torch + sentence-transformers 约 2GB，影响 Agent Service 的镜像大小和启动时间。
2. **独立扩缩容**：Embedding 和 Reranker 是 CPU/GPU 密集型，Agent 是 IO 密集型，分离后可独立扩容。
3. **模型替换不重启 Agent**：切换模型只需重启对应服务，不影响 Agent Service。
4. **设备隔离**：模型服务可以单独申请 GPU，Agent Service 不需要。

## 模型替换流程

### 替换 Embedding 模型

```bash
# 1. 修改 infra/.env
EMBEDDING_MODEL_NAME=BAAI/bge-m3
EMBEDDING_DIMENSION=1024
MILVUS_COLLECTION=local_life_kb_1024   # ← 必须新建，不能复用 768 维

# 2. 同步修改 copilot-agent-service/.env
EMBEDDING_MODEL_NAME=BAAI/bge-m3
EMBEDDING_DIMENSION=1024
MILVUS_COLLECTION=local_life_kb_1024

# 3. 重建 embedding-service
docker compose -f infra/docker-compose.dev.yml --profile rag up -d --build embedding-service

# 4. 重新入库（旧 collection 数据无效）
cd copilot-agent-service && python -m rag.ingest

# 5. 验证
curl http://localhost:8100/health   # 确认 "dimension": 1024
```

### 替换 Reranker 模型

```bash
# 1. 修改 infra/.env
RERANKER_MODEL_NAME=cross-encoder/ms-marco-MiniLM-L6-v2

# 2. 重建 reranker-service
docker compose -f infra/docker-compose.dev.yml --profile rag up -d --build reranker-service

# 3. 验证（Agent Service 无需重启）
curl http://localhost:8101/health
```

## Milvus 维度保护

`vector_store.py` 在以下时机检查维度一致性：

| 时机 | 检查内容 | 失败时行为 |
|------|----------|-----------|
| 服务启动连接 Milvus | collection 实际维度 vs `EMBEDDING_DIMENSION` | 抛 `DimensionMismatchError`，打印建议 |
| `upsert` 写入 | 向量长度 vs `EMBEDDING_DIMENSION` | 抛 `DimensionMismatchError` |

错误信息示例：
```
DimensionMismatchError: Milvus collection 'local_life_kb' 的向量维度为 768，
但当前配置 EMBEDDING_DIMENSION=1024（模型：BAAI/bge-m3）。
解决方案：
  1. 新建 collection：修改 MILVUS_COLLECTION 为新名称（如 local_life_kb_1024）
  2. 或恢复原模型：将 EMBEDDING_MODEL_NAME 改回 768 维模型
  3. 如需迁移：先删除旧 collection，再重新 ingest 所有文档
```

## 环境变量一览（必须在两个 .env 中保持一致）

| 变量 | infra/.env | copilot-agent-service/.env |
|------|-----------|---------------------------|
| `EMBEDDING_MODEL_NAME` | 控制 embedding-service 加载哪个模型 | 控制 Milvus collection 元数据校验 |
| `EMBEDDING_DIMENSION` | 控制 embedding-service 维度声明 | 控制 Milvus schema 维度 |
| `MILVUS_COLLECTION` | - | 控制使用哪个 collection |
| `RERANKER_MODEL_NAME` | 控制 reranker-service 加载哪个模型 | 不感知（调用 HTTP 接口） |
