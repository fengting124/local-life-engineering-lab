固定一套可以长期迭代的本地开发基线，保证后续文档、代码、容器编排、构建链路和调试链路都围绕同一组版本推进。

## 3. 为什么 JDK 选 17

- 当前项目主栈是 `Spring Boot 3.5.x`
- Java 后端求职场景里，`17` 兼顾稳定性、生态兼容性和面试接受度
- 本项目重点在工程问题，不在追逐更高语言版本

## 4. 本机状态结论

1. 当前默认 `java` 指向 `JDK 25`
2. `mvn` 缺失

这两个问题都会直接影响后续脚手架生成、构建和 IDE 导入。
