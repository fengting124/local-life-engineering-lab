## 关键面试答话模板（调研结论，直接用）

**被问「这是学习项目还是公司项目」**：
> 「个人项目，但底座是真实的本地生活交易平台业务系统，有完整的秒杀（Redis Lua原子操作）、支付（Outbox模式）、订单（ShardingSphere分表）等生产级模块。AI Copilot层是在这个真实业务底座上构建的，工具查询的是真实业务数据，不是Mock数据。」

**被问「为什么需要Cross-Encoder Reranker，向量相似度不够吗」**：
> 「Bi-Encoder 把查询和文档分开编码，相似度是近似的；Cross-Encoder 把查询+文档拼在一起输入模型，联合编码更精确，但不能预计算（只适合精排小集合）。我们的做法是先用 Milvus 向量搜索召回 Top20（快），再用 Cross-Encoder 精排到 Top5（准）。测下来 Recall@5 从 73% 提升到 85%+。」

**被问「为什么做并行工具执行」**：
> 「ReAct 循环中第2步往往需要同时调查多个方面（比如同时查支付状态和券发放日志），这两个工具调用互相独立，没有数据依赖。串行执行是 t1+t2，并发执行是 max(t1,t2)，对3步循环的任务理论上可以降低到 1/N 的延迟。实测 P99 从 18.5s 降到约 7s。」

**被问「MCP 和 Skills 区别」**：
> 「MCP（Model Context Protocol）是工具标准化协议，定义了 Agent 和工具之间的通信方式（JSON-RPC 2.0），解决工具集成标准化问题（类比 USB-C）。Skills 是编码特定领域能力和规范的知识单元，解决 Agent 的领域知识问题。MCP 是「怎么调」，Skills 是「知道什么」，两者互补。」
