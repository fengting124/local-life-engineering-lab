## 项目内容

是一个本地生活交易的软件，分为两部分，local life server 和 copiliot，


### 项目边界

- 项目边界不清的时候会导致模块膨胀，接口设计失焦，文档与代码脱节，且主线模糊

本项目的两个系统：
- LocalLife 就是一个面向本地生活场景的内容交易平台后端系统
- LocalLife Copilot 则是面向与商家运营和平台客服的 AI 辅助应用

两条线共享同一业务域，同一数据底座和事件流

#### 业务域定义
1. 内容社区
2. 交易系统
3. 商家运营
构成了真实业务闭环

#### 用户目标
1. C端客户
核心行为：
- 浏览门店
- 浏览摊探店容（帖子）
- 关注达人与商家
- 参与领券与秒杀
- 下单与支付
- 评论互动

2. B端商家
- 维护门店资料
- 发布活动和券
- 查看订单
- 查看内容运营反馈
- 查询经营数据

3. 平台运营与客服
- 排查订单异常
- 查询活动规则
- 解释券投放限制
- 借助 AI 工具完成知识问答和问题定位  

#### LocalLife Copilot 的业务边界
- 商家知识问答
- 活动配置辅助
- 订单异常排查
- 券规则解释
- 门店数据查询
- 内容运营分析辅助

重点：
- 工具调用
- 工作流控制
- rag
- 评测
- 流式返回
- MCP 暴露

### 核心业务对象
10个：用户 User ; 商家 ： Merchant；门店 shop；内容笔记 Post；评论 Comment； 关注 FollowRelation；优惠券模板 CouponTemplate ； 秒杀活动 SeckillSession；订单 Order； 支付单 paymentorder。
- 交易订单和支付单的区别：订单时交易主单，记录：谁买的，多少钱，进行到哪一步；支付单时支付过程单，表示怎么付钱，服了多少，是否成果，回调结果是什么。


### 四条核心主链路是什么
1. 登录链路
用户身份识别后进入平台，关注：
- 登录方式
- token 管理
- Redis 会话存储
- 基础权限边界

2. 发笔记链路
- 内容对象结构
- 图片资源占位
- 内容状态流转
- 评论与互动扩展点

3. 抢券链路
- 库存预扣减
- 一人一单限制
- redis 原子操作
- 异步下单与补偿

4. 下单支付链路
- 订单状态流转
- 支付结果处理
- 超时取消
- 对账补偿
