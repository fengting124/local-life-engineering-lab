# LocalLife 平台规则手册

> 适用范围：所有商家、客服、运营 | 权限范围：public

---

## 一、优惠券规则

### 1.1 券模板类型

平台支持两种优惠券类型：

- **现金券（CASH）**：直接抵扣固定金额。例如「满 50 减 10 元」。
- **折扣券（PERCENT）**：按订单金额折扣。例如「8 折券」表示订单金额 × 0.8。

### 1.2 使用门槛

每张券可以设置 `min_order_amount`（最低使用门槛，单位：分）。
- 0 表示无门槛，任意金额都可使用。
- 大于 0 时，订单金额必须 ≥ 门槛才能使用。

### 1.3 库存限制

券模板有两个库存字段：
- `total_stock`：总发行量。
- `remaining_stock`：当前剩余可领取数量。
- 当 `remaining_stock = 0` 时，状态自动变为 SOLD_OUT，不可再领取。

### 1.4 一人一张限制

同一用户对同一券模板只能领取一张，重复领取会返回 COUPON_ALREADY_RECEIVED 错误。
这是为了防止刷券行为，所有商家活动券都强制启用此规则。

### 1.5 过期规则

券有两个过期时间：
- 券模板的活动期 `end_time`：超过后不能再领取，但已领取的券仍可使用。
- 单张券的 `expire_at`：通常为领取后 30 天，超过后状态变为 EXPIRED。

## 二、订单规则

### 2.1 订单状态机

订单状态流转：
```
WAIT_PAY → PAID → COMPLETED
   ↓        ↓
CANCELLED  REFUNDING
```

- **WAIT_PAY**：待支付，订单创建后默认状态，30 分钟内必须支付。
- **PAID**：已支付，等待核销。
- **COMPLETED**：已核销，订单完成。
- **CANCELLED**：已取消，可由用户主动取消或超时自动关闭。
- **REFUNDING**：退款中（暂未上线，预留）。

### 2.2 支付超时

下单后 30 分钟内必须完成支付，超过时间订单自动关闭（CANCELLED），并归还优惠券。
后台定时任务每分钟扫描一次。

### 2.3 退款规则

- 已支付订单可以退款，退款金额 ≤ 实付金额。
- 退款必须经过人工审批（HITL），不允许 AI 或脚本自主执行。
- 退款成功后 3-5 个工作日到账（依支付渠道而定）。

## 三、商家规则

### 3.1 入驻资质

商家入驻平台需要：
1. 营业执照（已过工商注册）
2. 法人身份证
3. 食品经营许可证（餐饮类必需）
4. 银行对公账户（用于结算）

### 3.2 门店上下线

- 商家可主动下线门店（OFFLINE 状态），下线后顾客无法搜索到。
- 平台审核不通过时也会强制下线。
- 重新上线需要重新提交资质审核。

### 3.3 结算周期

- T+7：订单完成 7 天后自动结算到对公账户。
- 平台抽佣比例：餐饮 5%、零售 3%、服务类 8%。
- 抽佣金额在结算时自动扣除。

## 四、活动规则

### 4.1 活动审批

- 商家发布活动前需要平台审核（48 小时内）。
- 大促活动（双 11、618 等）需要提前 7 天报备。
- 国庆、春节等节日活动需要遵守平台节日营销规范。

### 4.2 活动数据要求

- 每个活动必须设置明确的开始时间和结束时间。
- 活动期间不允许临时修改券规则（库存除外）。
- 活动数据每天 0 点更新一次统计。

### 4.3 限购规则

每个活动可以设置「单用户限购」：
- 默认无限制。
- 可选择「每用户每天最多 N 单」或「整个活动周期最多 N 单」。

| 退款申请审批 | 24 小时内 |
| 高风险动作（HITL）审批 | 24 小时内（超时自动拒绝）|
| 活动审核 | 48 小时内 |
